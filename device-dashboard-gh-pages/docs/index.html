Device Dashboard Documentation

Welcome to the Device Dashboard project! This documentation provides an overview of the project, its structure, and how to use it.

Overview

The Device Dashboard is designed to analyze call quality metrics for Teams calls. It provides insights into various performance metrics, including video and audio quality, packet loss, jitter, and latency.

Getting Started

To get started with the Device Dashboard, you can clone the repository and open the index.html file in your browser.

Project Structure

  • index.html: Main entry point for the dashboard.
  • README.md: General project documentation.
  • .gitignore: Specifies files to be ignored by Git.
  • CNAME: Custom domain for GitHub Pages.
  • .github/workflows/deploy-gh-pages.yml: GitHub Actions workflow for deployment.
  • docs/index.html: Documentation entry point.
  • docs/README.md: Documentation for the docs directory.

Usage

Open index.html in your web browser to view the dashboard. You can interact with the filters and view various metrics and charts related to call quality.
